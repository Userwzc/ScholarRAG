# Task 5: Structured Citation and Provenance Data

## Implementation Summary

Extended the backend query flow to emit structured citation/provenance data with the following fields:
- `pdf_name`: Paper identifier
- `page`: Page number (int)
- `type`: Chunk type (text/image/table)
- `chunk_id`: Unique chunk identifier (evidence_id)
- `paper_version`: Version number (None until Task 6)
- `heading`: Section heading
- `supporting_text`: Truncated text excerpt (max 200 chars)

## Files Modified

1. **api/schemas.py** - Extended `SourceSchema` with new optional fields:
   - `chunk_id: Optional[str]`
   - `paper_version: Optional[int]`
   - `heading: Optional[str]`
   - `supporting_text: Optional[str]`

2. **src/agent/evidence_builder.py** - Added `build_structured_provenance()` function:
   - Builds structured provenance from evidence items
   - Deduplicates by (pdf_name, page, type) tuple
   - Truncates supporting text to 200 chars
   - Handles missing/invalid fields gracefully

3. **src/agent/graph.py** - Updated `run_agent_loop_events()`:
   - Collects ToolMessages during agent loop
   - Builds provenance from collected evidence
   - Includes `sources` in `answer_done` event

4. **api/services/query_service.py** - No changes needed:
   - Already passes events through with `json.dumps(event)`
   - `sources` field is automatically serialized

## Backward Compatibility

- Existing `pdf_name`, `page`, `type` fields remain required
- New fields are optional (None by default)
- Frontend can continue using existing fields
- New fields available for enhanced citation UI
