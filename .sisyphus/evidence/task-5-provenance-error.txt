# Task 5: Empty Sources for Insufficient Evidence

## Scenario: Unsupported question does not fabricate citations

When a query is issued that is unrelated to indexed papers, the system should:
1. Report insufficient evidence
2. Return no bogus `chunk_id`/page references
3. Return an empty `sources` array

## Implementation

The `build_structured_provenance()` function handles this correctly:

1. **Empty evidence input** → Returns empty list `[]`
2. **Evidence without pdf_name** → Skipped (not included in output)
3. **No tool calls made** → Empty `all_tool_messages` list → Empty provenance

## Test Coverage

```python
def test_empty_evidence_returns_empty_list(self) -> None:
    """Empty evidence should return empty provenance list."""
    result = build_structured_provenance([])
    assert result == []

def test_missing_pdf_name_skipped(self) -> None:
    """Evidence without pdf_name should be skipped."""
    evidence = [
        {
            "page_idx": 1,
            "chunk_type": "text",
            "text": "Content",
        }
    ]
    result = build_structured_provenance(evidence)
    assert len(result) == 0
```

## Example: Query with No Relevant Results

When the agent determines evidence is insufficient:

```json
{
  "type": "answer_done",
  "sources": []
}
```

The agent's response will indicate insufficient evidence without fabricating citations:

```
"I don't have sufficient evidence in the indexed papers to answer this question."
```
