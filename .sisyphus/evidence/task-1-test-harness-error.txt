### Observations:
- Tests still run using test overrides/fakes
- No auth/model-path crash occurs during collection
- The `test_env` fixture (autouse=True, session scope) sets mock values before any tests run
- Even with OPENAI_API_KEY and EMBEDDING_MODEL unset, the fixture provides:
  - `OPENAI_API_KEY=test-key-mock`
  - `EMBEDDING_MODEL=mock-model`

### How It Works:
1. The `test_env` fixture has `autouse=True` and `scope="session"`
2. It runs automatically before any tests are collected
3. It sets all required environment variables to mock values
4. Tests can safely import config modules without crashes
5. Original environment values are restored after the session

### Key Design Decisions:
- Session-scoped fixture ensures env is set before any imports
- Mock values are clearly identifiable (e.g., "mock-model", "test-key-mock")
- No real API keys or model paths are required
- Tests are deterministic and reproducible
